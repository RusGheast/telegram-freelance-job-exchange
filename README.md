# 📦 Бот для управления заказами в Telegram

Телеграм-бот для распределения заказов между пользователями с системой баланса, выполнением заданий и выводом заработанных средств. Администратор публикует заказы, пользователи берут их в работу, получают оплату на внутренний счёт и выводят деньги.

---

## ✨ Возможности

- **Пользовательская часть**
  - Просмотр профиля (баланс, количество выполненных заказов)
  - Список доступных заказов с возможностью взять в работу
  - Отказ от взятых заказов (до выполнения)
  - Запрос на вывод средств (сумма и реквизиты)
  - Удобная клавиатура с основными действиями

- **Административная часть**
  - Публикация новых заказов (название + описание)
  - Подтверждение заявок на вывод (встроенные кнопки)
  - Просмотр статистики (планируется, но в коде уже заложена структура)

- **Технические особенности**
  - Асинхронный движок (aiogram 3.x + SQLAlchemy 2.0 asyncio)
  - SQLite для хранения данных (легко заменить на PostgreSQL)
  - FSM для управления состояниями диалога
  - Инлайн-клавиатуры для действий администратора

---

## 🛠 Технологии

- Python 3.10+
- [aiogram](https://docs.aiogram.dev/) — асинхронный фреймворк для ботов
- [SQLAlchemy](https://www.sqlalchemy.org/) — ORM для работы с БД (async)
- [aiosqlite](https://github.com/omnilib/aiosqlite) — драйвер для SQLite
- [dotenv](https://github.com/theskumar/python-dotenv) — управление переменными окружения
- [Pydantic](https://docs.pydantic.dev/) — валидация конфигурации

## 🚀 Установка и запуск

1. **Клонируйте репозиторий**
    ```bash
   git clone https://github.com/RusGheast/telegram-freelance-job-exchange.git
   ```
   ```bash
   cd telegram-freelance-job-exchange
   ```

Создайте виртуальное окружение и активируйте его

```bash
python -m venv venv
source venv/bin/activate      # для Linux/Mac
venv\Scripts\activate         # для Windows
```

Установите зависимости

```bash
pip install aiogram SQLAlchemy aiosqlite dotenv Pydantic
```
Создайте файл .env в корне проекта и укажите:

```env
BOT_TOKEN=ваш_токен_бота
ADMIN_ID=ваш_telegram_id
ADMIN_ID — это числовой идентификатор пользователя, которому будут доступны административные функции.
```

Запустите бота

```bash
python bot.py
```
При первом запуске автоматически создадутся все необходимые таблицы в базе данных bot.db.

🧑‍💻 Команды бота
Доступные команды (настраиваются в handlers/user.py и handlers/admin.py):

/start — приветственное сообщение, главное меню

/menu — показать главное меню (для пользователей)

/admin — панель администратора (только для ADMIN_ID)

Остальное управление осуществляется через кнопки клавиатуры.
